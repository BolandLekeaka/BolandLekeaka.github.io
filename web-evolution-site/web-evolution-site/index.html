The Role and Evolution of HTTP

From HTTP/0.9 to HTTP/3 over QUIC—how the web’s application protocol keeps the internet usable, fast, and secure.

Key Terms

  • HTTP – Application-layer protocol for web communication.
  • Stateless – Each request is independent; no built-in memory of prior requests.
  • HTTP/2 – Adds multiplexing, header compression, and server push.
  • HTTP/3 – Runs over QUIC (UDP) to reduce latency and HoL blocking.
  • TLS 1.3 – Modern encryption for confidentiality and integrity.

Abstract

HTTP has powered the World Wide Web since the early 1990s. Its stateless, human-readable design made it easy to adopt and extend; modern versions (HTTP/2 and HTTP/3) improve performance, security, and reliability for rich applications, streaming, and global-scale platforms.

Why HTTP still matters

It’s the common language of browsers, servers, APIs, and even IoT devices.

Performance evolution

HTTP/2 introduces multiplexing; HTTP/3 moves to QUIC to avoid transport-level bottlenecks.

Security by default

TLS 1.3 and encrypted-by-default deployments make modern HTTP safer for users.
