HTTP History & Core Design

Early HTTP (0.9–1.0) enabled simple retrieval of documents and images. HTTP/1.1 standardized persistent connections and caching directives but remained stateless, meaning each request is processed without remembering prior ones. Developers layered cookies and sessions to deliver shopping carts, logins, and personalization.

Request Methods

  • GET – Retrieve a resource.
  • POST – Submit data for processing (e.g., forms).
  • PUT/PATCH – Replace/modify an existing resource.
  • DELETE – Remove a resource.

Challenges with HTTP/1.1

Heavy pages require many requests. With one outstanding request per TCP connection, sites suffered head-of-line blocking, domain sharding hacks, and slow loads.
