Key Concepts

  1. Statelessness – Each request is independent; state is added via cookies, sessions, or tokens.
  2. Methods – GET, POST, PUT, PATCH, DELETE map actions to resources identified by URLs.
  3. Headers – Metadata controlling caching, content negotiation, auth, and security.
  4. HTTP/2 Multiplexing – Multiple parallel streams on one connection to reduce latency.
  5. QUIC & HTTP/3 – UDP-based transport with integrated TLS 1.3 to mitigate head-of-line blocking.
  6. Caching – Directives (e.g., Cache-Control, ETag) improve performance and offload servers.
  7. Security – TLS 1.3 and HSTS enforce confidentiality/integrity and safer defaults.
